# 관리자 페이지 계획 (Draft)

## 목표
- 비개발자가 포트폴리오 콘텐츠를 쉽게 수정할 수 있는 관리자 페이지 구축
- 이미지/텍스트/링크 등을 폼으로 관리
- 배포된 사이트와 연동 시에도 안전하게 운영 가능

## 전제
- 현재 배포 방식: Cloudflare Pages + GitHub 연결
- 기존 구조: 정적 HTML/CSS/JS
- 관리 대상: `index.html`, `gallery.html` 등

## 관리 대상 후보
- 갤러리 카드(썸네일, 제목, 카테고리)
- 갤러리 상세 모달(설명, 제작일, 수정일, 링크)
- 타임라인 항목
- 친구 배너(이름, 링크, 배너 이미지)
- 공지/상태 보드 문구

## 운영 방식 옵션

### 옵션 A: 로컬 전용 관리자
- 로컬에서만 실행
- 수정 결과를 JSON으로 저장 → 정적 페이지가 JSON을 읽어 렌더링
- 장점: 간단, 보안 부담 낮음
- 단점: 배포된 페이지에서 직접 수정은 불가

### 옵션 B: 배포 사이트에서도 관리
- Cloudflare Pages + KV/DB 사용
- 관리자 페이지에서 입력 → KV에 저장 → 사이트가 KV 데이터를 읽음
- 장점: 배포 후에도 업데이트 가능
- 단점: 인증/보안/운영 복잡도 증가

## UI 구성 초안
- 왼쪽: 섹션 탭(갤러리/타임라인/친구배너/공지)
- 오른쪽: 폼(텍스트 입력, 이미지 경로, 날짜 입력)
- 저장 버튼 → JSON 갱신

## 필요한 질문
1. 관리자 페이지에서 수정할 항목 범위
2. 저장 방식(로컬/배포)
3. 로그인 필요 여부
4. 운영 환경(로컬만/배포 포함)
5. 이미지 업로드 필요 여부
6. 수정 대상 페이지 범위
